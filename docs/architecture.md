# Architecture

## Overview

React SPA connecting to a FastAPI BFF (Backend for Frontend). This repo is the UI only.
The BFF handles all communication with downstream services — the UI never calls downstream APIs directly.

## Layer Map

Dependencies flow strictly in one direction. Nothing skips a layer.

```
openapi.json              ← BFF contract, committed to repo, source of truth
     ↓
src/types/api.ts          ← generated by npm run gen:types, never hand-written
     ↓
src/lib/api-client.ts     ← base fetch wrapper, auth headers, error handling
     ↓
src/features/*/           ← TanStack Query hooks (queryFn / mutationFn only)
     ↓
src/components/           ← UI components, consume hooks, no direct API calls
src/routes/               ← TanStack Router pages, compose features and components
```

### Rules enforced by this layering

- `src/types/api.ts` is auto-generated — never edit it directly
- `src/lib/api-client.ts` is the only place raw `fetch` is called
- TanStack Query hooks live in `src/features/<feature>/hooks/` — not inside components
- Components and routes are not allowed to import from `src/lib/api-client.ts` directly
- Components and routes are not allowed to call `fetch` directly

Violations of these rules are flagged by `/audit` and blocked by CI.

## Folder Structure

```
src/
  components/          # shared, reusable UI components (no feature-specific logic)
  features/            # feature modules — self-contained vertical slices
    <feature>/
      components/      # feature-specific UI
      hooks/           # TanStack Query hooks for this feature
      index.ts         # public API — only export from here
  routes/              # TanStack Router route files
  hooks/               # shared custom hooks (non-query)
  lib/
    api-client.ts      # base fetch wrapper
    utils.ts           # cn() and other shared utilities
  types/
    api.ts             # GENERATED — do not edit
```

## Key Libraries and Their Roles

| Library | Role |
|---|---|
| TanStack Router | File-based routing, type-safe params |
| TanStack Query | All server state, caching, loading/error states |
| shadcn/ui | UI primitive components — extend, don't modify |
| Tailwind CSS | All styling — utility classes only |
| Zod | Runtime validation of API responses at the boundary |
| Vitest + RTL | Unit and component tests |
| msw | API mocking in tests |

## BFF Contract

See [bff-contract.md](./bff-contract.md) for the full OpenAPI flow and how to update types
when the BFF changes.

## Adding a New Feature

1. Add the feature to `features.json` before starting
2. Create `src/features/<feature>/` folder
3. Define API types via BFF — never hand-write them
4. Build hooks first, then components, then wire into routes
5. Export only through `src/features/<feature>/index.ts`
6. No other feature may import from inside your feature folder directly
